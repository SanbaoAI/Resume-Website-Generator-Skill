# Bilingual (i18n) Rules

Optional **zh / en** language switching for generated resume websites. Vanilla JS only — no i18n libraries.

---

## When to Enable

Enable bilingual mode when **any** of the following is true:

- User explicitly asks for 中英文切换 / bilingual / English version
- Resume is primarily Chinese but candidate targets international roles
- Resume includes both Chinese and English content
- Example or demo sites in this Skill repository (always enable for public demos)

Skip i18n when the user wants a single-language site only.

---

## Architecture

```
website/scripts/
├── i18n.js       # SITE_I18N dictionary + applySiteLanguage + initSiteI18n
└── main.js       # Calls initSiteI18n() on DOMContentLoaded
```

Load order in `index.html`:

```html
<script src="scripts/i18n.js"></script>
<script src="scripts/main.js" defer></script>
```

## Dictionary Structure

```javascript
window.SITE_I18N = {
  zh: {
    meta: { title: '...', description: '...' },
    skip: '跳转到主要内容',
    nav: { work: '项目', /* ... */ },
    hero: { taglineHtml: '...<span class="hero__accent">...</span>', /* ... */ },
    // mirror page sections: work, approach, experience, about, contact, footer
  },
  en: {
    meta: { title: '...', description: '...' },
    // full parallel translation — do not leave UI strings untranslated
  }
};
```

Rules:

- **Nested dot-path keys** — e.g. `hero.stat1`, `experience.e1role`
- **`meta.title` / `meta.description`** — update `<title>`, `<meta name="description">`, and Open Graph tags
- **HTML fragments** — use `data-i18n-html` for strings containing markup (accent spans, `<strong>`)
- **Plain text** — use `data-i18n`
- **aria-label** — use `data-i18n-aria` on sections, nav, buttons
- **Keep technical terms** — skill tags like `RAG`, `Agent`, repo names may stay English in both locales
- **Names** — translate or transliterate per locale (e.g. 林澈 ↔ Lin Che); brand names (SanbaoAI) stay unchanged

## HTML Markup

### Language toggle (nav)

Desktop — compact pill next to primary CTA:

```html
<div class="lang-toggle" role="group" data-i18n-aria="lang.group">
  <button type="button" class="lang-toggle__btn is-active" data-lang="zh" aria-pressed="true">中</button>
  <button type="button" class="lang-toggle__btn" data-lang="en" aria-pressed="false">EN</button>
</div>
```

Mobile menu — full labels:

```html
<div class="lang-toggle lang-toggle--mobile" role="group" data-i18n-aria="lang.group">
  <button type="button" class="lang-toggle__btn is-active" data-lang="zh" aria-pressed="true">中文</button>
  <button type="button" class="lang-toggle__btn" data-lang="en" aria-pressed="false">English</button>
</div>
```

### Translatable nodes

```html
<p data-i18n="hero.subhead">...</p>
<h1 data-i18n-html="hero.taglineHtml">...</h1>
<section id="work" data-i18n-aria="work.label">...</section>
```

Default HTML content = **primary locale** (usually `zh-CN`). English is applied on load if saved preference or browser language is English.

## JavaScript API

Required exports on `window`:

| Function | Purpose |
|----------|---------|
| `applySiteLanguage(lang)` | Apply `'zh'` or `'en'`; update DOM, meta, toggle state, `localStorage` |
| `initSiteI18n()` | Read saved lang or browser default; bind toggle buttons |

```javascript
document.documentElement.lang = lang === 'zh' ? 'zh-CN' : 'en';
localStorage.setItem('site-lang', lang);
```

Initial language priority: `localStorage('site-lang')` → browser `navigator.language` → default `zh`.

## CSS

Add to `components.css` (see `examples/example-output-sanbao/styles/components.css`):

- `.lang-toggle` — pill container, matches nav height (`--nav-item-height`)
- `.lang-toggle__btn` / `.lang-toggle__btn.is-active`
- `.lang-toggle--mobile` — full width in mobile nav drawer
- Light-theme overrides under `.nav__mobile .lang-toggle`

Do not use a separate CSS file for i18n.

## Quality Checklist

- [ ] Every visible UI string has a dictionary entry in **both** locales
- [ ] Toggle updates `<html lang>` dynamically
- [ ] Preference persists across reloads
- [ ] Mobile and desktop toggles stay in sync
- [ ] External links (GitHub, mailto) unchanged by locale
- [ ] No flash of wrong language on first paint (acceptable: brief; optimize if noticeable)
